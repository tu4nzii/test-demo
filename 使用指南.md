# 图表智能分析系统使用指南

## 系统概述

本系统是一个图表智能分析工具，支持处理和分析多种类型的图表，包括玫瑰图（Rose）、雷达图（Radar）以及直角坐标系图表（柱状图、折线图、散点图、气泡图等）。系统通过前端上传图表图片和数据，后端进行处理和评估，最终返回分析结果。

## 系统架构

### 1. 后端架构

后端使用 Python + FastAPI 框架构建，主要包含以下模块：

| 模块 | 功能 | 位置 |
|------|------|------|
| 主服务 | 提供API接口，处理前端请求 | `backend/main.py` |
| 图表类型检测 | 自动识别图表类型（支持9种类型） | `backend/function_call/chart_type.py` |
| 图表处理器 | 根据图表类型执行相应处理 | `backend/function_call/chart_processor.py` |
| 玫瑰图处理 | 专门处理玫瑰图的功能 | `backend/demo_rose/` |
| 雷达图处理 | 专门处理雷达图的功能 | `backend/demo_radar/` |
| 直角坐标系处理 | 处理柱状图、折线图、散点图等 | `backend/Grid_generation/` |

### 2. 前端架构

前端使用 Vue 3 + TypeScript + Vite 构建，主要包含以下模块：

| 模块 | 功能 | 位置 |
|------|------|------|
| 主应用 | 提供用户界面和交互逻辑 | `frontend/chart-demo-ui/src/App.vue` |
| API 交互 | 与后端API进行通信 | `frontend/chart-demo-ui/src/App.vue` |
| 状态管理 | 管理应用状态 | `frontend/chart-demo-ui/src/App.vue` |

## 支持的图表类型

系统目前支持以下 9 种图表类型：

| 类型 | 描述 | 坐标系 |
|------|------|--------|
| rose | 玫瑰图 | 极坐标 |
| radar | 雷达图 | 极坐标 |
| v_bar | 垂直条形图 | 直角坐标 |
| h_bar | 水平条形图 | 直角坐标 |
| line | 折线图 | 直角坐标 |
| scatter | 散点图 | 直角坐标 |
| bubble | 气泡图 | 直角坐标 |
| donut | 环形图 | 极坐标 |
| pie | 饼图 | 极坐标 |

## 快速开始

### 1. 环境准备

#### 后端环境

1. 进入后端目录：
   ```bash
   cd backend
   ```

2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

#### 前端环境

1. 进入前端目录：
   ```bash
   cd frontend/chart-demo-ui
   ```

2. 安装依赖：
   ```bash
   npm install
   ```

### 2. 启动服务

#### 启动后端服务

```bash
# 在 backend 目录下执行
python main.py
```

后端服务将在 `http://127.0.0.1:8000` 启动。

#### 启动前端服务

```bash
# 在 frontend/chart-demo-ui 目录下执行
npm run dev
```

前端服务将在 `http://127.0.0.1:5173` 启动。

### 3. 使用流程

1. **上传文件**：在前端界面上传图表图片和对应的 JSON 数据文件
2. **加密处理**：点击"加密处理"按钮，系统会对图表进行处理
3. **评估预测**：点击"进行预测"按钮，系统会对处理后的图表进行评估并返回结果

## 核心功能说明

### 1. 图表类型检测

系统会自动检测上传的图表类型，支持 9 种图表类型检测：
- 使用 LLM（大语言模型）进行图表类型识别
- 返回图表类型和置信度
- 支持 rose、radar、v_bar、h_bar、line、scatter、bubble、donut、pie

### 2. 图表加密处理

直角坐标系图表处理流程：
- **直线检测**：使用 Hough 变换检测图表中的直线
- **坐标轴推断**：识别 X 轴和 Y 轴的位置和方向
- **刻度线检测**：检测坐标轴上的刻度线位置
- **刻度标签识别**：使用 LLM 识别刻度值，支持数值轴和文字轴
- **颜色提取**：使用 LLM 提取图表的主要颜色
- **网格生成**：生成基础网格和加密网格（插入中间刻度值）

### 3. 图表评估

- 对处理后的图表进行评估
- 生成详细的评估结果
- 提供数据可视化

## API 接口说明

### 1. 上传文件接口

- **URL**: `/api/upload/`
- **方法**: POST
- **参数**:
  - `file`: 图片文件
  - `json_data`: JSON 数据文件
- **返回**:
  - `chart_id`: 图表唯一标识符
  - `chart_type`: 图表类型
  - `confidence`: 检测置信度

### 2. 处理图表接口

- **URL**: `/api/process/`
- **方法**: POST
- **参数**:
  - `chart_id`: 图表 ID（查询参数）
- **返回**:
  - `encrypted_image_url`: 加密后图片的 URL 路径

### 3. 评估图表接口

- **URL**: `/api/evaluate/`
- **方法**: POST
- **参数**:
  - `chart_id`: 图表 ID（查询参数）
- **返回**:
  - `results_url`: 评估结果的 URL 路径

### 4. 获取图片接口

- **URL**: `/api/images/{filename}`
- **方法**: GET
- **返回**: 图片文件

### 5. 获取结果接口

- **URL**: `/api/results/{filename}`
- **方法**: GET
- **返回**: 评估结果 JSON

## 常见问题与解决方案

### 1. 后端服务启动失败

**问题**：运行 `python main.py` 时出现错误

**解决方案**：
- 检查是否安装了所有依赖：`pip install -r requirements.txt`
- 检查端口 8000 是否被占用
- 检查 Python 版本是否为 3.8+

### 2. 前端服务启动失败

**问题**：运行 `npm run dev` 时出现错误

**解决方案**：
- 检查是否安装了所有依赖：`npm install`
- 检查 Node.js 版本是否符合要求（^20.19.0 || >=22.12.0）
- 检查端口 5173 是否被占用

### 3. 上传文件失败

**问题**：点击"上传"按钮后出现错误

**解决方案**：
- 检查文件格式是否正确（图片支持 png、jpeg 格式）
- 检查 JSON 文件格式是否正确
- 检查后端服务是否正常运行

### 4. 处理图表失败

**问题**：点击"加密处理"按钮后出现错误

**解决方案**：
- 确保已成功上传文件并获得 chart_id
- 检查图表类型是否被正确识别
- 检查后端日志获取详细错误信息
- 对于饼图、环形图等无坐标轴图表，处理失败是预期行为

### 5. 评估预测失败

**问题**：点击"进行预测"按钮后出现错误

**解决方案**：
- 确保已成功处理图表并获得加密后的图片
- 检查后端日志获取详细错误信息

1. 在 `backend/function_call/chart_type.py` 中添加新的图表类型检测逻辑

- 优化图像处理算法，减少处理时间
- 可以添加结果缓存避免重复处理

## 直角坐标系图表处理详解

### 处理流程

1. **图像读取**：支持中文路径的图像读取
2. **直线检测**：使用 Hough 变换检测图表中的所有直线
3. **直线合并**：将相似角度和位置的直线合并
4. **坐标轴推断**：根据直线位置和角度推断 X 轴和 Y 轴
5. **刻度线检测**：在坐标轴上检测刻度线的位置
6. **刻度过滤**：过滤掉噪声和非刻度线
7. **标签识别（LLM）**：使用大模型识别刻度标签值
8. **颜色提取（LLM）**：使用大模型提取图表的主要颜色
9. **网格生成**：
   - 基础网格：绘制坐标轴和原始刻度线
   - 加密网格：在原始刻度之间插入中间值

### 轴类型支持

系统支持两种轴类型：
- **数值轴**：刻度为纯数字，如 1, 2, 3.5, -4 等
- **文字轴**：刻度为文字、字母或年份、月份等，如 Jan, Feb, 2020 等

对于数值轴，系统会生成加密刻度（在原始刻度之间插入中间值）；
对于文字轴，保持原样不加密。

### 输出文件

对于每个成功处理的图表，系统会生成：
- **{chart_id}_grid.png**：基础网格图像
- **{chart_id}_with_grid.png**：加密网格图像
- **{chart_id}_ticks.json**：刻度信息，包含：
  - `x_ticks`：X 轴原始刻度值
  - `y_ticks`：Y 轴原始刻度值
  - `x_ticks_encrypted`：X 轴加密刻度值
  - `y_ticks_encrypted`：Y 轴加密刻度值
  - `x_axis_type`：X 轴类型
  - `y_axis_type`：Y 轴类型
  - `colors`：图表颜色信息
